## 배경 (Background)

현재 가격 체계는 **정적·단순**합니다.

| 도메인 | 현 가격 구조 | 한계 |
|---|---|---|
| `Slot` (시설 예약) | **price 필드 없음** — `ownerId`, 시간만 보유 | 시설 예약 유료화·시간대별 차등 불가 |
| `Seat` (티켓) | `price: BigDecimal` 고정 | 수요·잔여석 기반 변동가 불가 |
| `Product` (굿즈) | `price: BigDecimal` 고정 | 할인·세일·번들 불가 |
| `Payment` | `amount` 그대로 청구 | 쿠폰·프로모션 적용 단계 없음 |

→ 결과: **수요가 몰리는 주말 황금시간대도 동일가, 비수기 유인책도 없고, 쿠폰 마케팅 수단이 전무**합니다.

### 비즈니스 기회

| 레버 | 효과 |
|---|---|
| 시간대·요일 변동가 | 피크 수익 극대화 + 비피크 수요 유도 |
| 잔여석 기반 티켓 변동가 | 매진 임박 시 프리미엄, 저조 시 할인 |
| 쿠폰/할인코드 | 신규 유치·재방문·제휴 마케팅 |
| 기간 프로모션 | 시즌·이벤트 캠페인 |

본 PRD는 결제 직전 단계에 **가격 산정 레이어**를 삽입합니다. 기존 결제 엔진(멱등·Saga)은 그대로 두고, "청구 금액을 무엇으로 정할지"만 정책화합니다.

## 기능 요구사항 (Functional Requirements)

### FR-01. Slot 가격 필드 도입

**결과**:
- `Slot`에 `basePrice: BigDecimal` 추가 (nullable → 기본 0/무료 호환)
- 예약 생성 시 Payment 연동 (현재 무료 예약 → 유료 옵션). 0원 예약은 결제 스킵 호환
- 마이그레이션: 기존 slot `basePrice = 0` 백필 (하위 호환)

> 기존 무료 예약 플로우를 깨지 않도록 0원은 결제 우회. 가격 > 0일 때만 결제 진입.

### FR-02. 가격 정책 엔진 (PricingRule)

**결과**:
- `PricingRule` 엔티티: 대상(slot/event/product 범위), 조건(요일·시간대·잔여율 임계), 조정(정률 %/정액), 우선순위, 유효기간
- 정책 적용 순서: basePrice → 매칭되는 rule들 우선순위순 적용 → 조정가
- Seat(티켓) 변동가: 잔여석 비율 기반 (예: 잔여 <20% → +15%, >80% → -10%)
- 정책은 **결정적**: 같은 (대상, 시각, 잔여) → 같은 결과

<details>
<summary>PricingRule 적용 예시</summary>

```
basePrice 10,000
+ rule[주말] +30%  → 13,000
+ rule[저녁 18-21시] +10% → 14,300
최종 단가 14,300 (쿠폰 적용은 FR-04에서)
```

</details>

### FR-03. 쿠폰/할인코드 도메인

**결과**:
- `Coupon` 엔티티: code(unique), 할인(정률/정액), 최소주문액, 사용한도(전체/1인), 유효기간, 대상범위(도메인/상품)
- `CouponRedemption` 매핑 엔티티 (**@ManyToMany 금지**): coupon_id, user_id, payment_id, redeemed_at — 1인 사용한도·중복 차단
- 발급 방식: 공개코드 / 타겟 발급(특정 user)
- 검증: 만료·한도·최소금액·대상 일치

### FR-04. 결제 시점 가격 산정 (Quote)

**결과**:
- 결제 전 `PriceQuote` 산정: 정가 → PricingRule 적용 → 쿠폰 차감 → 최종 청구액
- Quote에 적용 내역(라인 아이템) 포함 — 사용자 투명 표시
- **Quote 멱등**: 같은 입력 + 같은 시점은 동일 Quote (Payment 멱등키와 연계)
- 결제 시 Quote의 최종액으로 Payment.amount 확정 — 가격 조작 차단(서버 산정)

### FR-05. 프로모션 캠페인

**결과**:
- `PromotionCampaign`: 기간, 대상(전체/세그먼트), 연결된 쿠폰/룰 묶음, 활성 토글
- 캠페인 시작/종료 스케줄 자동 적용
- 성과 추적: 캠페인별 사용·매출 기여 → 인사이트 위젯

### FR-06. 운영자 가격/쿠폰 관리 UI + MCP tool

**결과**:
- 포털: 가격 정책 CRUD, 쿠폰 발행·조회, 캠페인 관리
- MCP write tool 신규: `createPricingRule`, `issueCoupon`, `startCampaign` — confirm flow 재사용
- AI 어시스턴트(C2) 연계: "비수기 감지 → 할인 캠페인 제안" 가능
